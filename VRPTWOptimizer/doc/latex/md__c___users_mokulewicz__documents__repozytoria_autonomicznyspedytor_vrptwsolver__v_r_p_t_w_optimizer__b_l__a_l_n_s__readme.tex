ALNS algorithm is based on removal and insertion heuristics. Algorithm accepts as an input an initial solution (generated here using Nearest Neighbor heuristic). The general idea of the algorithm is to use removal operator to destroy part of the current solution and then insertion heuristic to repair it and construct new solution.

In the current implementation Shaw removal heuristic is used. It removes requests from the solution based on the relatedness measure. The relatedness measure defines the similarity of two requests by comparing locations, service start times, capacities and the vehicles that can be used to serve the two requests.

In order to construct new solutions after removal procedure two types of heuristics are used\+: basic greedy heuristic and regret heuristics. Basic greedy heuristic chooses among unplanned requests the one that minimizes the cost of inserting request at its best position overall and inserts this request at its minimum cost position. This process continues until all requests have been inserted or no more requests can be inserted.

The regret heuristic tries to improve upon the basic greedy heuristic by incorporating a kind of look ahead information when selecting the request to insert. In the regret heuristic a regret value is defined as the difference in the cost of inserting the request in its best route and its second best route. In each iteration the regret heuristic chooses to insert the request that maximizes the reget value. This heuristic can be extended in a natural way to define a class of regret heuristics\+: the regret-\/k heuristic.
