Adaptive Large Neighborhood Algorithm

Algorithm description

ALNS algorithm is based on removal and insertion heuristics. Algorithm accepts as an input an initial solution (generated here using Nearest Neighbor heuristic). The general idea of the algorithm is to use removal operator to destroy part of the current solution and then insertion heuristic to repair it and construct new solution.

In the current implementation Shaw removal heuristic is used. It removes requests from the solution based on the relatedness measure. The relatedness measure defines the similarity of two requests by comparing locations, service start times, capacities and the vehicles that can be used to serve the two requests.

In order to construct new solutions after removal procedure two types of heuristics are used: basic greedy heuristic and regret heuristics. Basic greedy heuristic chooses among unplanned requests the one that minimizes the cost of inserting request at its best position overall and inserts this request at its minimum cost position. This process continues until all requests have been inserted or no more requests can be inserted.

The regret heuristic tries to improve upon the basic greedy heuristic by incorporating a kind of look ahead information when selecting the request to insert. In the regret heuristic a regret value is defined as the difference in the cost of inserting the request in its best route and its second best route. In each iteration the regret heuristic chooses to insert the request that maximizes the reget value. This heuristic can be extended in a natural way to define a class of regret heuristics: the regret-k heuristic. Heuristics with k > 2 investigate the cost of inserting a request on the k best routes and insert the request whose cost difference between inserting it into the best route and the kâˆ’1 best routes is largest. In the current implementation k=2,3,4 or k is the total number of routes in the current solution.

ALNSGetData.cs

Contains functions that create input data for heuristics: list of delivery nodes, fleet, times and distances.

ALNSHeuristics.cs

Contains implementation of all heuristics used in main ALNS algorithm.

ALNSNode.cs

Represents a request.

ALNSRoute.cs

ALNSRoute objects consist of the set of ALNSNodes in the order they are visited, ALNSVehicle used to serve all the nodes, route total length and travel time, TripNumber corresponds to a number of a race performed by the vehicle serving the route, and total load is a sum of demands.

ALNSSolution.cs

Consisists of ALNSRoutes and set of ALNSNodes that are not visited.

It is used as an input and ouput of heuristics.

ALNSVehicle.cs

Represents vehicle and its properties.

Contains auxiliary functions to identify all possible vehicles that can serve delivery nodes. They are used in Shaw removal heuristic in relatedness measure function.

ALNSVRPTWOptimizer.cs

Contains ALNS algorithm.
